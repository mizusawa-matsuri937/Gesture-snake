# Gesture Snake

基于 Python、pygame、OpenCV 和 MediaPipe 的手势控制贪吃蛇游戏。

本版本的核心玩法是：用食指指尖作为目标点，蛇头会平滑追随食指移动。蛇不会瞬间贴到手指位置，而是按当前速度自然靠近目标点，以减少手抖和快速移动造成的断节。

## 主要功能

- 开始菜单：单人模式和闯关模式可玩，双人模式显示“开发中 / Coming Soon”。
- 食指追随控制：食指指尖移动到哪里，蛇头就平滑追向哪里。
- 选项设置：主菜单中可调整手势灵敏度，让小范围手指移动控制全地图。
- 捏合点击：食指指尖和大拇指指尖捏合用于点击菜单按钮。
- 和平手势：Game Over 后主动重开当前模式。
- 单手锁定：只锁定第一只进入画面的手，忽略后出现的第二只手。
- 单人无尽模式：右侧游戏区穿墙循环，只有撞到自己身体才会结束。
- 闯关模式：有目标分数和墙壁障碍，撞墙、撞边界、撞身体都会失败。
- 普通苹果和限时红色大苹果：单人模式中每吃 5 个普通苹果会出现一次 5 秒限时大苹果。
- 流畅度优化：使用 dt 时间步、缓存网格背景、连续轨迹采样和平方距离碰撞检测。

## 安装与运行

需要 Python 3.8+ 和摄像头。

安装依赖：

```bash
pip install -r requirements.txt
```

运行游戏：

```bash
python snake.py
```

游戏默认以全屏运行。按 `F11` 可在全屏和窗口模式之间切换，按 `ESC` 退出。

在当前 Windows 项目环境中，推荐直接使用项目自带虚拟环境：

```bash
.\.venv\Scripts\python.exe snake.py
```

如果你的系统里 `python` 指向 Python 2，请改用 Python 3 解释器运行，例如：

```bash
python3 snake.py
```

## 操作方式

| 操作 | 效果 |
| --- | --- |
| 移动食指指尖 | 蛇头平滑追随食指目标点 |
| 食指和大拇指捏合 | 点击菜单、暂停、结束页面按钮 |
| 主菜单 `Options` | 调整手势灵敏度 |
| 和平手势 | Game Over 后重新开始当前模式 |
| 鼠标点击 | 备用点击方式，方便测试 |
| ESC | 退出游戏 |

只有锁定的手完全丢失超过短暂时间后，游戏才会进入追踪丢失暂停状态；重新识别到手后会自动继续。和平手势不会在游玩中暂停。

## 菜单说明

- `Single Player`：已完成，可进入无尽模式。
- `Level Mode`：已完成，可进入带墙壁和目标分数的关卡。
- `Duo Mode`：开发中，当前显示 `Coming Soon`。
- `Options`：可调整手势灵敏度。

## 单人无尽模式规则

1. 蛇在右侧主游戏区移动，左侧摄像头栏不属于游戏区域。
2. 撞到边界不会死亡，会从另一侧穿出。
3. 唯一死亡条件是蛇头撞到自己的身体。
4. 开局有约 3 秒保护时间，避免初始身体重叠误判。
5. 普通苹果一直存在：
   - 得分 +10
   - 蛇身轻微增长
   - 普通苹果重新生成
   - 苹果计数 +1
6. 每吃 5 个普通苹果会生成一个红色大苹果：
   - 只存在 5 秒
   - 得分 +50
   - 蛇身增长更多
   - 与普通苹果不会重叠生成
7. 速度规则：
   - 基础速度：180 像素/秒
   - 每个普通苹果增加：8 像素/秒
   - 最大速度：420 像素/秒

## 闯关模式规则

1. 蛇、苹果和墙壁都只出现在右侧主游戏区。
2. 撞左边界、右边界、上边界、下边界都会 Game Over；左边界是侧边栏右侧，不是窗口最左侧。
3. 撞墙和撞到自己身体都会 Game Over。
4. 每关开始时蛇会回到安全出生点，长度和目标点重置。
5. 每关有目标分数，达到目标后进入“关卡完成 / Level Clear”状态。
6. Level Clear 约 1 秒后自动进入下一关。
7. 当前实现至少包含 5 个关卡；第 1 关无墙用于适应，第 2 关开始出现墙壁。
8. 闯关模式苹果使用辅助网格生成，避免出现在墙里、蛇身附近、离蛇头太近或明显死路里。

闯关模式速度规则：

- 基础速度：160 像素/秒
- 每吃一个苹果增加：4 像素/秒
- 最大速度：300 像素/秒
- 普通苹果得分：10
- 普通苹果增长：1

## 手势识别逻辑

MediaPipe Hands 最多检测 2 只手，但游戏只使用第一只进入画面的手：

- 第一次检测到手时，将它锁定为 active hand。
- 后续出现第二只手时，不会切换控制权。
- 锁定手短暂丢失时，不会马上切换到另一只手。
- 锁定手丢失超过约 1 秒后，才允许重新锁定新的第一只手。

控制数据只来自锁定手：

- 食指指尖：MediaPipe landmark 8
- 大拇指指尖：MediaPipe landmark 4
- 捏合点击：landmark 8 和 4 的归一化距离低于阈值时触发
- 和平手势：食指和中指伸出、无名指和小指收起时触发

摄像头画面的完整区域都可用于手势判定，不再需要把手放进黄色框。

## 手势灵敏度

主菜单中有 `Options`，可以调整手势灵敏度：

- Normal：`x1`
- Sensitive：`x3`
- High：`x4`
- Ultra：`x6`

灵敏度会围绕摄像头画面中心放大食指位移。默认使用 `High x4`，这样手不需要移动到摄像头边缘，也能让蛇追向地图边缘。

## 游戏状态

- `MENU`：开始菜单，选择 Single Player / Level Mode / Duo Mode。
- `OPTIONS`：从主菜单进入，调整手势灵敏度。
- `PLAYING_SINGLE`：单人无尽模式。
- `PLAYING_LEVEL`：闯关模式。
- `LEVEL_CLEAR`：关卡完成提示，约 1 秒后自动进入下一关。
- `PAUSED`：暂停状态。只有手丢失才会触发追踪暂停，重新识别到手后自动继续。
- `GAMEOVER`：显示最终信息，可按钮重开或和平手势重开。
- `COMING_SOON`：双人模式预留页面。

## 常见问题

### 中文显示成方框

程序会优先加载 Windows 中文字体文件，例如微软雅黑、黑体、宋体或 Noto Sans SC。如果仍出现方框，请确认 `C:\Windows\Fonts` 中存在可用中文字体。

### 摄像头打不开

请检查：

- 摄像头是否被其他程序占用。
- 系统是否允许 Python 访问摄像头。
- 笔记本或外接摄像头是否正常连接。

程序会在终端输出清晰的摄像头错误提示。

### 调整摄像头分辨率

程序默认请求更宽的 16:9 摄像头画面，并在预览区按比例显示完整画面，不再裁剪成正方形。在 `config.py` 顶部修改这些设置即可：

```python
CAMERA_INDEX = 0
CAMERA_RESOLUTION = (1280, 720)
CAMERA_FPS = 30
```

如果机器性能或摄像头不支持 `(1280, 720)`，可以降低到 `(960, 540)` 或 `(640, 480)`。代码无法改变摄像头物理镜头视角，但会尽量使用完整画面并保持较高默认灵敏度，减少手出界的概率。

### 蛇身断节

蛇身使用连续轨迹采样。快速移动食指时，程序会在蛇头路径上补充中间身体点，避免出现大块空隙。

### 捏合连续触发

捏合点击带有防抖和冷却。只有从“未捏合”变为“捏合”的瞬间才算一次点击，持续捏合不会连续点击。
