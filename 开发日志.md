# Gesture Snake 开发日志

日期：2026-06-02  
项目：Gesture Snake 手势控制贪吃蛇  
技术栈：Python、pygame、OpenCV、MediaPipe、NumPy

## 一、开发目标

本阶段的目标是把原始“手势控制贪吃蛇”从简单的手部追踪演示，整理成一个更完整、可玩的单人无尽模式游戏版本。

重点包括：

- 增加正式开始菜单。
- 实现单人无尽模式。
- 修复手势控制逻辑和蛇身运动体验。
- 增加捏合点击、和平手势重开、单手锁定等交互。
- 增加可调手势灵敏度，让小范围手指移动也能控制全地图。
- 优化中文显示、蛇身连续性、摄像头配置和 README 文档。

## 二、主要开发内容

### 1. 游戏状态机

当前游戏已整理为多个明确状态：

- `MENU`：开始菜单。
- `OPTIONS`：选项设置页面，用于调整手势灵敏度。
- `PLAYING_SINGLE`：单人无尽模式。
- `PAUSED`：追踪丢失暂停。
- `GAMEOVER`：游戏结束页面。
- `COMING_SOON`：闯关模式和双人模式的预留页面。

开始菜单中目前包含：

- 单人模式：可进入正式游戏。
- 选项设置：可调整手势灵敏度。
- 闯关模式：显示开发中。
- 双人模式：显示开发中。

### 2. 手势识别与交互

手势识别由 `VisionSystem` 负责，主要输出：

- 是否检测到锁定手。
- 食指指尖归一化坐标。
- 捏合点击事件。
- 和平手势事件。
- 摄像头预览帧。

当前交互规则：

- 食指指尖用于控制蛇头目标位置。
- 食指和大拇指捏合用于点击按钮。
- 和平手势只在 `GAMEOVER` 状态用于重新开始。
- 游戏中不再通过和平手势暂停。
- 只有锁定手丢失超过短暂时间后才进入暂停。

为了避免第二只手抢控制权，单人模式只锁定第一只进入画面的手。后续出现的第二只手不会切换控制权。

### 3. 蛇的运动方式

蛇的运动经历了两次重要调整：

1. 先从“蛇头直接追随手的位置”改为“食指只改变方向，蛇自动前进”。
2. 根据实际体验反馈，又改为当前版本的“食指目标点平滑追随”。

当前逻辑是：

- 食指指尖坐标会映射到右侧游戏区。
- 蛇头以当前速度平滑靠近目标点。
- 蛇头不会瞬间贴住手指，避免手抖导致跳跃。
- 蛇身使用连续轨迹采样，快速移动时会补中间身体点，避免出现严重断节。

### 4. 手势灵敏度设置

为了让玩家不需要把手移动到摄像头边缘，也能控制蛇覆盖全地图，新增了手势灵敏度设置。

灵敏度会围绕摄像头画面中心放大食指位移：

- 标准：`x1`
- 灵敏：`x3`
- 高灵敏：`x4`
- 超高：`x6`

默认值为高灵敏。玩家可以在主菜单的“选项设置”中调节。

### 5. 单人无尽模式规则

当前单人模式规则：

- 蛇在右侧游戏区域移动。
- 撞墙不会死亡，会从另一侧穿出。
- 唯一死亡条件是撞到自己的身体。
- 开局有短暂保护时间。
- 普通苹果常驻，吃到后加分并轻微增长。
- 每吃 5 个普通苹果会出现一个限时红色大苹果。
- 大苹果只存在 5 秒，分值更高，增长更多。
- 蛇的最大追随速度会随着苹果数量增加，并有最大速度限制。

当前增长参数：

- 普通苹果增长：`NORMAL_GROWTH = 1`
- 大苹果增长：`BIG_GROWTH = 3`

### 6. 中文字体修复

开发过程中发现 pygame 默认字体可能无法渲染中文，导致 UI 中文显示为方框。

当前修复方式：

- 优先从 Windows 字体目录加载真实中文字体文件。
- 支持微软雅黑、黑体、宋体、Noto Sans SC 等字体。
- 字体加载失败时再回退到系统字体或 pygame 默认字体。

### 7. 摄像头设置

当前摄像头设置集中在 `snake.py` 顶部：

```python
CAMERA_INDEX = 0
CAMERA_RESOLUTION = (640, 480)
CAMERA_FPS = 30
```

这些设置可以直接修改，用于调整摄像头索引、分辨率和目标帧率。

## 三、问题与修复记录

### 中文显示成方框

原因：pygame 默认字体缺少中文字形。  
修复：增加真实中文字体文件优先加载逻辑。

### 蛇身严重断节

原因：蛇头移动距离较大时，每帧只插入一个身体点，相邻身体点之间距离过大。  
修复：改为沿蛇头路径插入多个中间点，保证身体连续。

### 普通苹果增长过多

原因：原增长值较大，吃一个苹果后蛇身增长过明显。  
修复：普通苹果增长降为 1，大苹果增长降为 3。

### 和平手势误触暂停

原因：和平手势曾同时承担游玩中暂停和 Game Over 重开的功能。  
修复：移除和平手势暂停，只保留 Game Over 后重开。

### 手势控制范围不够

原因：食指坐标直接映射摄像头全画面，玩家需要移动较大幅度才能覆盖游戏地图。  
修复：新增手势灵敏度倍率，在选项菜单中可调。

### Python 环境差异

发现 Miniconda Python 3.12 环境中的 MediaPipe 不提供 `mp.solutions`，但项目 `.venv` 的 Python 3.10 环境可正常使用 MediaPipe Hands。

当前建议运行命令：

```bash
.\.venv\Scripts\python.exe snake.py
```
